# 🌊 Workflow: Идея → MVP

## 1. Фаза "Идея" (💡 Idea)
1. Создать файл в `00 Входящие` по шаблону `01 Idea`.
2. Заполнить проблему и гипотезы.
3. Провести быстрый поиск аналогов.

## 2. Фаза "Проект" (🏗️ Project)
1. Если идея прошла проверку, создать папку `01 Проекты/[Название]`.
2. Создать файл `[Название].md` по шаблону `02 Project Master`.
3. Заполнить `idea_ref`.

## 3. Фаза "Проектирование" (🏛️ Arch & 🎨 Brand)
1. В папке проекта создать `Architecture.md` (шаблон `04 Architecture`) и `Brand.md` (шаблон `05 Brand`).
2. Определить Tech Stack на базе `Standards.md`.

## 4. Фаза "Декомпозиция" (✅ Tasks)
1. JARVIS сканирует Architecture и Brand.
2. Создаются файлы `Feature.md` (шаблон `03 Feature`) в папке `Tasks`.
3. Каждая фича получает приоритет.

## 5. Фаза "Код" (💻 Repo)
1. JARVIS генерирует структуру репозитория в `Architecture.md`.
2. Реализация фич по одной.

## 6. Фаза "MVP" (🚀 Launch)
1. Проверка чек-листа DoDone.
2. Деплой и сбор обратной связи.
